# Latency eval — 2026-06-17

**Setup:** run from in-region EC2 (`us-east-1`), so network distance is removed and these reflect
the engines, not the laptop→internet hop. Turbopuffer + Pinecone are managed (in-region); Qdrant
is Qdrant Cloud (`us-east-1`). Consistency = **strong** (Turbopuffer; Pinecone always eventual),
**no** prewarm. 5 queries, both collections. `topK`/`iters` swept via `run.sh`.

All numbers below are **end-to-end** latency in **ms** — both collections queried in parallel
(`max` of the two), which is what an app experiences per search. Embedding latency is excluded
(shared across services). Filter = `session_id == 2163 AND notification_action_time >= 2026-06-10`.

> `iters=5` → 25 calls/service (noisy tail); `iters=50` → 250 calls (trustworthy). Weight the
> 50-iteration rows.

## Observations

- **Correctness ✓** — every run reported identical filter hits (`bill_text`=1, `bill_amendment`=0)
  across all three services, so the `session_id` + epoch filter maps the same everywhere.
- **Turbopuffer** is the steadiest: p50 ~16–23ms across all combos with the tightest tails. Strong
  consistency (its default here) didn't blow it up.
- **Pinecone** has the best *median* on filtered queries (p50 ~10–11ms) but the **worst tails** —
  repeated max spikes of 280–510ms. Great typical case, unreliable worst case.
- **Qdrant** shows a recurring **~53ms plateau** on one collection per run (the other ~5–11ms),
  which inflates its end-to-end despite a low ~5ms floor — looks like cold-segment/connection
  warmup variance on Qdrant Cloud, not steady-state compute. The cleanest run (topK=50, iters=50
  unfiltered) had qdrant **fastest** (avg 18.9, p50 17). Re-running with `--warm` would likely
  collapse that plateau.
- **topK barely matters** in this range — going 5→50 adds only a few ms everywhere; network +
  per-query overhead dominate over the extra neighbors.

> Caveat: even in-region, Qdrant Cloud and the two managed services aren't perfectly identical
> (instance sizing, multi-tenancy, warmup state differ). Treat gaps under ~10ms as noise.
